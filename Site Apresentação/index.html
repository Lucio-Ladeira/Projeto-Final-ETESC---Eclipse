APRESENTAÇÃO DO PROJETO

Esta é a ideia principal do ECLIPSE, o projeto final da ETESC em conjunto com Lúcio Ladeira, Davi Garrocho e Saymon da Silva

OBJETIVO

O projeto ECLIPSE tem como objetivo facilitar o acesso ao mundo da computação para pessoas com pouco ou nenhum conhecimento na área. Para isso, oferece um sistema capaz de montar automaticamente uma build personalizada com base nas respostas de um formulário preenchido pelo usuário. Além disso, também será possível criar builds manualmente, salvá-las e compará-las, proporcionando mais controle e autonomia na escolha.

JUSTIFICATIVA

O projeto surgiu a partir da percepção de que não existe, no mercado, um sistema como o nosso, o que dificulta significativamente o acesso ao mundo da computação. Entendemos que tanto o mercado quanto a vida cotidiana estão cada vez mais voltados para a tecnologia, tornando seu uso praticamente inevitável no futuro. Diante disso, reconhecemos que o acesso a esse universo é essencial e buscamos contribuir para tornar essa evolução mais acessível a todos.

REQUISITOS DO SISTEMA

REQUISITOS FUNCIONAIS

Sistema de Login

  • O sistema deve permitir que o usuário crie uma conta
  • O sistema deve permitir que o usuário logue com e-mail e senha
  • O sistema deve permitir recuperação de senha
  • O sistema deve permitir fazer o logout
  • O sistema deve permitir que o usuário escolha suas preferências para o sistema
  • O sistema deve permitir que o usuário insira os seus dados

Sistema de Vendas

  • O sistema deve permitir que o usuário coloque itens no carrinho
  • O sistema deve avisar a quantidade de estoque de algum item, e se tem algum estoque
  • O sistema deve validar e calcular um desconto por cupom
  • O sistema deve permitir avaliações
  • O sistema deve exibir uma opção de ver todos os itens de uma categoria em sua tela principal
  • O sistema deve permitir pagamento por pix, por cartão de crédito, e por paypal
  • O sistema deve permitir avaliações mostrar a descrição detalhada do produto, incluindo o desempenho estimado dele

Sistema de Buscas

  • O sistema deve filtrar por ordem alfabética
  • O sistema deve filtrar por ordem alfabética
  • O sistema deve filtrar por desempenho
  • O sistema deve filtrar por marca
  • O sistema deve filtrar por tipo de produto

Sistema de Entregas

  • O sistema deve receber o endereço para entrega
  • O sistema deve validar o pagamento antes de efetuar o envio do produto

Sistema de Montagem de PC

  • O sistema deve permitir a montagem de um computador personalizado
  • O sistema deve validar a compatibilidade das pessas ao adicionar elas na build
  • O sistema deve permitir que o usuário salve a build montada
  • O sistema deve permitir que o usuário compare as builds salvas por ele
  • O sistema deve mostrar o desempenho estimado da build
  • O sistema deve mostrar os jogos e softwares que o pc deve rodar
  • O sistema deve permitir o usuário escolher priorizar o desempenho, longevidade ou custo da build
  • O sistema deve perguntar quais softwares e games o usuário pretende usar
  • O sistema deve perguntar ao usuário quanto ele tem de orçamento
  • O sistema deve mostrar 3 opções de builds. A mais barata, a intermediária e a mais cara para o uso descrito pelo usuário
  • O sistema deve permitir que o usuário salve um setup como o setup possuido por ele, e então deve sugerir os melhores upgrades

Sistema de Suporte

  • O sistema deve possuir um chat com um agente de IA para suporte
  • O sistema deve possuir um e-mail para suporte personalizado

REQUISITOS NÃO FUNCIONAIS

  • O sistema deve ser seguro
  • O sistema deve permitir até 500 acessos simultâneos
  • O sistema deve proteger os dados de pagamento
  • O sistema deve ter tempo de resposta rápido (< 2ms)
  • O sistema deve ser estável
  • O sistema deve ter alta disponibilidade
